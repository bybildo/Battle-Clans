# Battle-Clans (Game Logic Engine)

**Статус проєкту: Завершено (Практика застосування патернів)**

Проєкт присвячений реалізації основної логіки ігрової сесії з фокусом на архітектурні патерни проектування GoF (Gang of Four). Основна мета — створення гнучкої та розширюваної системи керування ігровими об'єктами та станами.

## Технологічний стек
* **Мова програмування:** C# (.NET)
* **Парадигма:** Об'єктно-орієнтоване програмування (OOP)
* **Принципи:** SOLID

## Реалізовані патерни проектування
У проєкті практично застосовано наступні патерни для вирішення конкретних задач архітектури:

* **Singleton:** Забезпечення єдиної точки доступу до менеджера ігрової сесії.
* **Factory:** Ізоляція логіки створення різних типів ігрових юнітів.
* **Memento:** Реалізація системи збереження та відновлення стану ігрової сесії.
* **Decorator:** Динамічне додавання нових характеристик та можливостей об'єктам без зміни їхньої структури.
* **Mediator:** Зменшення зв'язності (coupling) між різними компонентами ігрової логіки через централізований об'єкт взаємодії.
* **State:** Керування поведінкою ігрових персонажів залежно від їхнього поточного стану (наприклад, HorseWarriorState).

## Особливості реалізації
* **Розділення відповідальностей:** Логіка поведінки, збереження та створення об'єктів рознесена по окремих модулях відповідно до принципів SOLID.
* **Гнучкість:** Завдяки використанню абстракцій, система дозволяє легко додавати нові типи воїнів або ігрові механіки без модифікації існуючого ядра.
* **Керування станами:** Використання патерну State дозволяє реалізувати складну поведінку юнітів, яка змінюється в процесі гри.
